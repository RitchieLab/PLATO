Filter Process Configuration

PLatform for the Analysis, Translation, and Organization of large-scale data (PLATO) incorporates several analytical and knowledge-based filters to help identify SNPs in a genome-wide association study.

All results from the filters can be scaled from 0 to 1. Filters can be applied sequentially to reduce the number of SNPs being analyzed. Results can be reduced to a manageable number or all results can be output for further investigation. Alternatively, all results can be kept and recorded.

Configuration file

The configuration file specifies the general parameters for the analysis and the filters that will be used to conduct the analysis. The basic format is KEYWORD VALUE. The keyword and values may be separated by spaces or tabs. The basic PLATO keywords are below. Filters have their own keywords for determining the parameters of the analysis. Comments can be placed in a configuration file and should begin with # (see examples).

Keyword Sample Value Description
ANALYSIS Establishes start of analysis. Required before filters listed.
LOCICOMBO ADD 1 Manages a list of SNPs and results that are passed from one filter to the next. This should be specified for each filter and should be placed before the START keyword that identifies the parameters for that filter. The options are: ADD - adds the indicated size of combination to the list of SNP combinations to analyze. REMOVE - removes the indicated size PASS - passes the current list of SNPs on to the next filter. Doesn't have to be specified as it is the default option.
START CHISQUARE Indicates the start of a filter. Filters will run in the order they are specified in the configuration file. All of the filter's parameters need to be placed before the END keyword for the filter.
END CHISQUARE Terminates the parameter list for a filter. All of the filter's parameters must be between the START and END keywords.
SUB MDR Indicates that this filter will be used as a subfilter by the enclosing filter. SERF is an example of a filter that uses a subfilter. A subfilter is terminated by an END keyword and otherwise has to follow the rules for parameter declaration.
RANDSEED 17 Random number generator seed. Used by some filters.
PSEUDOCONTROLS ON Creates a pseudocontrol for a pedigree input dataset consisting of trios.

Filters

PLATO contains multiple filters and each has its own parameters to control its behavior.
  1. Chisquare - Calculates chi square on SNPs. It can be used as a sub filter by other filters such as SERF, Contingency, and Biofilter
  2. LogisticRegress - Calculates logistic regression on the models. Parameters can be set so that multi-locus models can include interaction terms.
  3. MDR - Runs MDR on models. Doesn't include permutation testing.
  4. OddsRatio - Calculates odds ratio. Works on a contingency table and can be used by the contingency filter.
  5. NMI - Calculates NMI and can calculate transposed NMI. Also works on a contingency table and can be used by the contingency filter.
  6. Armitage - Calculates Armitage on a contingency table.
  7. LikelihoodRatio - Calculates likelihood ratio from a contingency table
  8. UncertaintyCoeff - Calculates uncertainty coefficient from a contingency table
  9. Contingency -Can use a number of filters as sub filters to perform a variety of calculations on contingency tables at one time so that the table only needs to be totaled once. The filters it can use are OddsRatio, Chisquare, NMI, Armitage, LikelihoodRatio and UncertaintyCoeff.
  10. AlleleFreq - Calculate major allele frequency for a SNP. Can be used on cases, controls or both.
  11. SERF - Runs sequential replication filter
  12. ConditionalLR - Runs conditional logistic regression on family data. Currently only works on trios or if the families have one affected and one unaffected child.
  13. MaxScore - Best score from sub filters and assigns a p value based on permutation testing.
  14. BioFilter - Selects models from a database and then uses subfilters to analyze the results.
  15. Tails - Jonathan's new method

Configuration file examples


1. Simple example that runs chi square on all SNPs 
ANALYSIS

# directive to the system specifying what size of
# locus combination to use
LOCICOMBO ADD 1

START CHISQUARE

# threshold for p-value to report to screen or pass
# to next filter
THRESHOLD 1.00

END CHISQUARE

# General parameters used in this analysis

2. Example shows use of a sub filter 
ANALYSIS

# directive to the system specifying what size of
# locus combination to use
LOCICOMBO ADD 1


START SERF
# This sub filter is used by sequential replication
# THRESHOLD needs to be 1.0 for sequential replication to work correctly
SUB CHISQUARE
THRESHOLD 1.0
END CHISQUARE

# Starting size of the group (50 affected and 50 unaffected)
INITSIZE 50
# Initial value returned from subfilter needed to count as a replication
INITTHRESHOLD 0.01
# Value that repeats after first much reach
REPTHRESHOLD 0.05
# Number of repeats required to pass this filter test
REPEATREQUIRED 0
# Optimizing parameter
# Entire set is not recalculated every time
# Only vaid with some sub filers such as
# the chi square filter
STARTOPT CHITYPE STARTCHI
OPTIMIZER CHITYPE ADDINDS

END SEQREP

# General parameters used in this analysis
RANDSEED 17

3. Configuration file showing analysis of single locus models using chi square followed by analysis of 1 and 2 locus models using MDR. 
ANALYSIS

# directive to the system specifying what size of
# locus combination to use
LOCICOMBO ADD 1

START CHISQUARE
# threshold for p-value to report to screen or pass
# to next filter
THRESHOLD  1.00
END CHISQUARE

LOCICOMBO ADD 2

START MDR

# threshold for balanced accuracy to report to screen or pass
# to next filter
ACCTHRESHOLD 0

# min size of combination to check
COMBOMIN 1
# max size of combination to check
COMBOLIMIT 2
# how to determine model threshold
# how to determine model threshold
# for assigning high and low risk cells
# use SET when no missing data
MODTHRESHCALC SET

END MDR

# General parameters used in this analysis

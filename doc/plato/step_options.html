Listing Step Options

Option: Description:
-all Override allele frequency overall calculation to use all samples. Cannot be used with -unaff-spouses-only, -unk-spouses, -random-child, or -all-children.

Valid on: hw, allele-freq, chisquare,
  ex: -all
-all-children Override allele frequency overall calculation to use all children. Cannot be used with -unaff-spouses-only, -unk-spouses, -random-child, or -all.

Valid on: hw, allele-freq, chisquare,
  ex: -all-children
-allele-custom Option for formatting steps to flip alleles based on custom mapping as specified in {filename}

Valid on: output-beagle, output-eigenstrat, output-fbat, output-grr, output-lapis, output-pdt2, output-phase, output-powermarker, output-structure, output-superlink, output-tped, output-qtdt, output-ped,
  ex: -allele-custom myallelemap.txt
-allele12 Converts genotype output to 1s and 2s. 1 refers to the minor allele, 2 refers to the major allele. Possible genotype codings include: 1 1, 1 2, 2 2, and 0 0 (for missing).

Valid on: output-beagle, output-eigenstrat, output-fbat, output-grr, output-lapis, output-pdt2, output-phase, output-powermarker, output-structure, output-superlink, output-tped, output-qtdt, output-ped,
  ex: output-ped -allele12
-allele1234 Option for formatting steps to flip from ACGT to 1234 alleles

Valid on: output-beagle, output-eigenstrat, output-fbat, output-grr, output-lapis, output-pdt2, output-phase, output-powermarker, output-structure, output-superlink, output-tped, output-qtdt, output-ped,
  ex: -allele1234
-alleleACGT Option for formatting steps to flip from 1234 to ACGT

Valid on: output-beagle, output-eigenstrat, output-fbat, output-grr, output-lapis, output-pdt2, output-phase, output-powermarker, output-structure, output-superlink, output-tped, output-qtdt, output-ped,

Valid on: deletions, gender-error, hw, homozygous, ld, marker-geno-eff, sample-geno-eff, allele-freq, chisquare, tdt, ibs, output-beagle, output-eigenstrat, output-fbat, output-grr, output-lapis, output-mdr, output-pdt2, output-phase, output-powermarker, output-structure, output-superlink, output-tped, cmh, mito-check, mendelian-error, mars, logreg, linear-reg, output-qtdt, output-ped, output-bin,

-dummy-incomplete-parent-ids This option will do one of two things when formatting output:

In the case of a Sample having two parent ids in the PED file and one of those parent ids has no genotypes listed in the file, the resulting formatted output will add that missing sample to the file and fill in with 0/0 genotypes.

In the case of a Sample having one parent id and the other one listed as missing (0), then the resulting output will fill in the 0 with DUM999999 for fathers and DUM999998 for mothers and add these dummy samples to the output filled in with 0/0 genotypes.

Valid on: output-qtdt, output-ped, output-bin,
  ex: output-ped -dummy-incomplete-parent-ids
-dummy-missing-parents Inserts pedigree info and 0/0 genotypes for parents that are in the childs Dad/Mom column but don't exist in the pedfile.

-strat-file Specifies a stratification file to be used when creating a STRUCTURE input file. The Structure documentation defines this field as the following: An integer designating a user-dened population from which the individual was obtained (for instance these might designate the geographic sampling locations of individuals). In the default models, this information is not used by the clustering algorithm, but can be used to help organize the output. Basically this is just a grouping of the Samples that you have into predefined population groups. If you are not sure how to define them, then just use a value of 1.

Columns required in file:
FamilyID
SampleID
Stratification value (usually numeric).


Example:
Fam123 100 1
Fam123 101 1
Fam124 100 1
Fam124 101 1
Fam223 100 2
Fam223 101 2
Fam862 1 3
Fam456 1 3
Fam231 1 3
SMP123 1 4
Fam324 1 4
...

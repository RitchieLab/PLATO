Listing commandlines

New commandline
Option: Description:
-S Prints list of valid steps that can be placed in the batch file.
  ex: plato -S
-bin-input {filename_prefix} Specifies Binary fileset to use.
  ex: plato batch -bin-input mybinaryfiles
-bp-max {number} Specifies a maximum chromosome specific base-pair location to load. All markers with base-pairs greater than the number will be removed. Can be used in conjunction with -bp-min and -bp-space. Note: -chrom is required.
  ex: plato batch -ped pedfile.ped -map mapfile.map -chrom 3 -bp-max 15000000
-bp-min {number} Specifies a minimum chromosome specific base-pair location to load. All markers with base-pairs less than the number will be removed. Can be used in conjunction with -bp-max and -bp-space. Note: -chrom is required.
  ex: plato batch -ped pedfile.ped -map mapfile.map -chrom 3 -bp-min 5000000
-bp-space {number} Specifies to load only markers spaced by {number} base-pairs across all chromosomes. Can be used in conjunction with -bp-min and -bp-max. Note: -chrom is not required.
  ex: plato batch -ped pedfile.ped -map mapfile.map -bp-space 1000000
-chrom {number} Specifies a specific chromosome to load. Only this one chromosome will be loaded and processed for all steps. (Note: X = 23, Y = 24)
  ex: plato batch -ped pedfile.ped -map mapfile.map -chrom 3 -bp-min 5000000
-dog Specifies the map and ped file are to be associated with Dog chromosomes. Chrom X is mapped to 39, Chrom Y is mapped to 40.
  ex: plato batch -ped pedfile.ped -map mapfile.map -dog
-excfamilies {filename} Specifies a list of families to exclude.
  ex: plato batch -ped pedfile.ped -map mapfile.map -excfamilies famexclusionlist.txt
-excmarkers {filename} Specifies a list of markers to exclude.
  ex: plato batch -ped pedfile.ped -map mapfile.map -excmarkers mrkexclusionlist.txt
-excsamples {filename} Specifies a list of samples to exclude.
  ex: plato batch -ped pedfile.ped -map mapfile.map -excsamples sampexclusionlist.txt
-flip {filename} Specifies a list of markers with strand information to flip to the opposite strand.
  ex: plato batch -ped pedfile.ped -map mapfile.map -flip newstrandinfo.txt
-freq-file {filename} Specifies a file containing predefined minor allele frequencies.
  ex: plato batch -ped pedfile.ped -map mapfile.map -freq-file frequencies.txt
-h Prints help. For most up to date help, please reference this website
  ex: plato -h
-incfamilies {filename} Specifies a list of families to include.
  ex: plato batch -ped pedfile.ped -map mapfile.map -incfamilies faminclusionlist.txt
-incmarkers {filename} Specifies a list of markers to include.
  ex: plato batch -ped pedfile.ped -map mapfile.map -incmarkers mrkinclusionlist.txt
-incsamples {filename} Specifies a list of samples to include.
  ex: plato batch -ped pedfile.ped -map mapfile.map -incsamples sampinclusionlist.txt
-make-bin Converts specified PED and Map files into binary format. No steps in the batch file will be processed.
  ex: plato batch -ped pedfile.ped -map mapfile.map -make-bin
-map {filename} Specifies Map file to use.
  ex: plato batch -ped mypedfile.ped -map mymapfile.map
-map-includes-ref Tells Plato that the map file contains 4 columns. The format is the same as the original map file, however a 4th column is added to specify which allele is the reference allele. For the 4th column, please specify only 1 allele for each entry.
  ex: plato batch -ped pedfile.ped -map mapfile.map -map-includes-ref
-mapdesc {filename} Specifies an optional extended MAP file to use.
  ex: plato batch -ped pedfile.ped -map mapfile.map -mapdesc extramapinfo.txt
-mdrmap Specifies the MAP file to go along with your -mdrped command. MAP file can be 3, 4, 5 or 6 columns depending on your needs. 3 column format is: Chrom, RSID, BPLOC. 4 column format is: Chrom, RSID, BPLOC, Referent_Allele. 5 column format is: Chrom, RSID, BPLOC, Minor_Allele, Major_Allele. 6 column format is: Chrom, RSID, BPLOC, Referent_Allele, Minor_Allele, Major_Allele.
  ex: plato batch -mdrped mymdrpedfile.ped -mdrmap mymdrmapfile.map
-mdrped Specify a MDR style ped file where the genotypes are encoded by a single variable 0, 1, 2, or -1 (for missing). This type of input is a slight diversion from the standard MDR input in that the first 6 columns of a standard PED file must exist along with the genotype encodings. No header line is required. If one exists, results are undefined. Requires a MAP file similar to the requirement for a standard PED file.
  ex: plato batch -mdrped mymdrpedfile.ped -mdrmap mymdrmapfile.map
-micro-sats Tells Plato to allow for markers with more than 2 alleles.
  ex: plato batch -ped mypedfile.ped -map mymapfile.map -micro-sats
-missing-geno {single_allele_of_genotype} Overrides the default missing genotype call (default is 0 0) with the user specified call. Please note that the value that is specified here is just a single allele of the missing genotype call. For example, if you want to specify the new missing call to be -9 -9, you would use the option like this: -missing-geno -9.
  ex: -missing-geno -9
-no-summary Disables marker, sample, family summary file creation at end of processing
  ex: plato batch -ped mypedfile.ped -map mymapfile.map -no-summary
-noweb Turns off automatic web based version/bug checking.
  ex: plato batch -ped pedfile.ped -map mapfile.map -dog
-out {prefix} Specifies a prefix to add to all output files.
  ex: plato batch -ped pedfile.ped -map mapfile.map -out test
-ped {filename} Specifies PED file to use.
  ex: plato batch -ped mypedfile.ped -map mymapfile.map -no-summary
-pedinfo {filename} Provide supplemental pedigree information that will override what was initially read in by Plato from the orignal input PED file.

File format is first 6 columns of the PED file.
  ex: plato batch -ped pedinput.ped -map pedmap.map -pedinfo pedigrees.txt
-print-families Once the ped file has been loaded, Plato will print to a file how it sees the familial relationships as it was specified in the pedfile by drawing a rough (very rough) tree diagram.
  ex: plato batch -ped pedfile.ped -map mapfile.map -print-families
-remaining Outputs a summary of the totals of the snps, samples, and families that are remaining after all the processing is done. Lists of items remaining are output to files prefixed with "Remaining".
  ex: plato batch -ped pedfile.ped -map mapfile.map -remaining
-sampdesc {filename} Specifies an optional extended Sample file to use.
  ex: plato batch -ped pedfile.ped -map mapfile.map -sampdesc extrasampleinfo.txt
-tfam {filename} Specifies TFAM file to use.
  ex: plato batch -tped mytpedfile.tped -tfam myfammap.tfam
-todigit Converts sample ids to numbers. A mapping file from the original names to the new digits will be created.
  ex: plato batch -ped pedfile.ped -map mapfile.map -todigit
-tped {filename} Specifies TPED file to use.
  ex: plato batch -tped mytpedfile.tped -tfam myfammap.tfam
-zero-geno-file {filename} All sample/snp pairs specified in the file will be turned into missing calls while the data is being processed. Format of the file specified is 3 columns:

FamID
IndID
SNP
  ex: plato batch -ped pedfile.ped -map mapfile.map -zero-geno-file pleasezerothese.txt

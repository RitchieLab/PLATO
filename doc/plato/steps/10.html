Logistic Regression

Performs logistic regression. Will not use covariates unless specified.


Batch file command:

logreg


Step specific options:
-boxcox-transform Performs the Box-Cox data transform.

Ex: tdt -boxcox-transform
-bp-max {number} Specifies a maximum chromosome specific base-pair location to process. All markers with base-pairs greater than the number will be excluded for the specified step only. Can be used in conjunction with -bp-min and -bp-space. Note: -chrom is required.

Ex: -chrom 3 -bp-max 15000000
-bp-min {number} Specifies a minimum chromosome specific base-pair location to process. All markers with base-pairs less than the number will be excluded for the specified step only. Can be used in conjunction with -bp-max and -bp-space. Note: -chrom is required.

Ex: -chrom 3 -bp-min 5000000
-bp-space {number} Specifies to process only markers spaced by {number} base-pairs across all chromosomes. Can be used in conjunction with -bp-min and -bp-max. Note: -chrom is not required.

Ex: -bp-space 1000000
-chrom {number} Specifies a specific chromosome to process. Only this one chromosome will be processed for the step specified. (Note: X = 23, Y = 24, XY = 25, Mitochondrial = 26)

Ex: -chrom 3
-ci {value} Specify confidence interval (default is 0.95 if flag is not specified)

Ex: -ci 0.80
-cond-lr-full-interact Same as -lr-full-interact, but for conditional analysis.

Ex: -cond-lr-full-interact
-cond-lr-inc-interact Same as -lr-inc-interact, but for conditional analysis

Ex: -cond-lr-inc-interact
-cond-lr-max-iter {value} Same as -lr-max-iter, but for conditional analysis.

Ex: -cond-lr-max-iter 30
-cond-lr-model-type {model} Same as -lr-model-type, but for conditional analysis.

Ex: -cond-lr-model-type DOMINANT
-covars-name {list of columns} Specifies named columns to use in covariate file separated by commas. Dashes are used to denote ranges of covariate columns.

Ex: -covars-name AGE-NUM_FOUND,GRE,FN
-covars-number {list of columns} Specifies column numbers in the covariate file to use separated by commas. Dashes denote range of covariate columns.

Ex: -covars-number 1-3,5,7
-lambda {number} When performing multiple comparisons, you can specify -lambda {number} to modify the Lambda value for the Box-Cox comparison.

Ex: chisquare -mult-compare -lambda 2
-log-transform Performs logistic data transform.

Ex: tdt -log-transform
-lr-full-interact Includes all interactions possible in the model passed to the method.

Ex: -lr-full-interact
-lr-inc-interact Interactions are included in the model.

Ex: -lr-inc-interact
-lr-max-iter {number} Maximum number of iterations that algorithm will perform before stopping. There are cases where the alogrithm doesn't converge on an answer and so this limits that problem. Default is 20.

Ex: -lr-max-iter 50
-lr-model-type {model} Specifies model type. Default is ADDITIVE. This option takes an argument and needs to be DOMINANT, RECESSIVE or ADDITIVE

Ex: -lr-model-type DOMINANT
-lr-no-interact No interactions are included. Only the main effects are evaluated

Ex: -lr-no-interact
-lr-reduced-interact Include all interactions except for the highest order interaction. This is a way to check the significance of the interaction as the user can run the full interaction and then the model without it.

Ex: -lr-reduced-interact
-mult-compare Enables multiple comparison outputs to be generated during analysis. The multiple comparisons that are performed include:

Genomic-control corrected p-values (GC)
Bonferroni single-step adjusted p-values (BONF)
Holm (1979) step-down adjusted p-values (HOLM)
Sidak single-step adjusted p-values (SIDAK_SS)
Sidak step-down adjusted p-values (SIDAK_SD)
Benjamini & Hochberg (1995) step-up FDR control (FDR_BH)
Benjamini & Yekutieli (2001) step-up FDR control (FDR_BY)

Ex: chisquare -mult-compare
-out {name} Overrides the suffix of the output files to be {name}

Ex: -out test1
-sqrt-transform Performs the square-root data transform

Ex: tdt -sqrt-transform

Mendelian Errors

The mendelian error step has two stages.

Stage 1: This looks at each mother, father, child goup and performs simple checks for mendelian error. Each error encountered by an individual increments the individual error count. Each family incurs only one error increment per marker. Each marker only incurs one error increment per family.

Stage 2: This stage emulates the level 2 error checking as seen in Pedcheck. The errors that occur here are displayed in a separate output file and are not added to the counts generated in stage 1. This is because some of the level 2 errors could be speculative as opposed to real error.

Thresholds for markers are based on total number of marker errors.
Thresholds for families are based on total number of family errors.


Batch file command:

mendelian-error


Step specific options:
-bp-max {number} Specifies a maximum chromosome specific base-pair location to process. All markers with base-pairs greater than the number will be excluded for the specified step only. Can be used in conjunction with -bp-min and -bp-space. Note: -chrom is required.

Ex: -chrom 3 -bp-max 15000000
-bp-min {number} Specifies a minimum chromosome specific base-pair location to process. All markers with base-pairs less than the number will be excluded for the specified step only. Can be used in conjunction with -bp-max and -bp-space. Note: -chrom is required.

Ex: -chrom 3 -bp-min 5000000
-bp-space {number} Specifies to process only markers spaced by {number} base-pairs across all chromosomes. Can be used in conjunction with -bp-min and -bp-max. Note: -chrom is not required.

Ex: -bp-space 1000000
-out {name} Overrides the suffix of the output files to be {name}

Ex: -out test1
-thresh-families-max {number} Specifies an upper cutoff threshold for families. All families with values greater than the specified value will be removed from future steps.

Ex: -thresh-families-max 100
-thresh-families-min {number} Specifies a lower cutoff threshold for families. All families with values less than the specified value will be removed from future steps.

Ex: -thresh-families-min 10
-thresh-markers-max {number} Specifies an upper cutoff threshold for markers. All markers with values greater than the specified value will be removed from future steps.

Ex: -thresh-markers-max 0.95
-thresh-markers-min {number} Specifies a lower cutoff threshold for markers. All markers with values less than the specified value will be removed from future steps.

Ex: -thresh-markers-min 0.95
-thresh-samples-max {number} Specifies an upper cutoff threshold for samples. All samples with values greater than the specified value will be removed from future steps.

Ex: -thresh-samples-max 0.95
-thresh-samples-min {number} Specifies a lower cutoff threshold for samples. All samples with values less than the specified value will be removed from future steps.

Ex: -thresh-samples-min 0.95
-zero Zeros out genotypes with mendelian errors.

Ex: -zero
-zero-l2 Zeros genotypes of all samples with level 2 genotype errors for the markers where the errors exist

Ex: -zero-l2
-zero-l2-fams Zero genotypes of all samples in a family for level 2 mendelian errors

Ex: -zero-l2-fams

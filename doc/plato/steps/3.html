Concordance

Performs a concordance check using the initial genotype information that is loaded when Plato starts against genotype files that are specified using flags for this step in the batch file. You can specify any file format that Plato is able to read. Plato will only compare samples across both sets of files with the same family ids and individual ids. Plato will iterate over all samples in the ped file specified in the batch file. It will compare each of these samples against EVERY match that it finds in the reference data set (ie: data set specified at run time).


Batch file command:

concordance


Step specific options:
-bin-input {fileprefix} Specifies Binary fileset to use.

Ex: -bin-input myprefix
-inc-missing Tells the concordance step to include missing genotypes in the calculations.

Ex: concordance -ped myped.ped -map mymap.map -inc-missing
-map {filename} Specifies Map file to use.

Ex: -ped myped.ped -map mymap.map
-out {name} Overrides the suffix of the output files to be {name}

Ex: -out test1
-ped {filename} Specifies PED file to use.

Ex: -ped myped.ped -map mymap.map
-tfam {filename} Specifies TFAM file to use.

Ex: -tped myped.tped -tfam myfam.tfam
-tped {filename} Specifies TPED file to use.

Ex: -tped myped.tped -tfam myfam.tfam
-unique-id Tells the concordance step that you are using the Father ID field in a ped file as a unique identifier for samples. Especially useful if there are multiple samples with the same Family ID and Ind IDs in a ped file.

Ex: concordance -ped myped.ped -map mymap.map -unique-id

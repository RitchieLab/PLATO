GRR

Creates a GRR input file or set of input files.


Batch file command:

output-grr


Step specific options:
-allele-custom {filename} Option for formatting steps to flip alleles based on custom mapping as specified in {filename}

Ex: -allele-custom myallelemap.txt
-allele12 Converts genotype output to 1s and 2s. 1 refers to the minor allele, 2 refers to the major allele. Possible genotype codings include: 1 1, 1 2, 2 2, and 0 0 (for missing).

Ex: output-ped -allele12
-allele1234 Option for formatting steps to flip from ACGT to 1234 alleles

Ex: -allele1234
-alleleACGT Option for formatting steps to flip from 1234 to ACGT

Ex: -alleleACGT
-bp-max {number} Specifies a maximum chromosome specific base-pair location to process. All markers with base-pairs greater than the number will be excluded for the specified step only. Can be used in conjunction with -bp-min and -bp-space. Note: -chrom is required.

Ex: -chrom 3 -bp-max 15000000
-bp-min {number} Specifies a minimum chromosome specific base-pair location to process. All markers with base-pairs less than the number will be excluded for the specified step only. Can be used in conjunction with -bp-max and -bp-space. Note: -chrom is required.

Ex: -chrom 3 -bp-min 5000000
-bp-space {number} Specifies to process only markers spaced by {number} base-pairs across all chromosomes. Can be used in conjunction with -bp-min and -bp-max. Note: -chrom is not required.

Ex: -bp-space 1000000
-chrom {number} Specifies a specific chromosome to process. Only this one chromosome will be processed for the step specified. (Note: X = 23, Y = 24, XY = 25, Mitochondrial = 26)

Ex: -chrom 3
-inc-disabled-samples Includes disabled samples in output of formatting steps to maintain pedigree information.

Ex: -inc-disabled-samples
-inc-excluded-samples Include in output excluded samples specified by the command line argument(s) -incsamples or -excsamples

Ex: -inc-excluded-samples
-out {name} Overrides the suffix of the output files to be {name}

Ex: -out test1
-override-out {name} Overrides the default output filename(s) with whatever is specified in {name}. Note: all filename extensions (.txt, .map, .ped, etc...) that Plato uses will be APPENDED to the name that is given.

Ex: output-ped -override-out post_qc
-random-markers {number} Choose random number of markers up to the number specified per set. Markers are NOT allowed to repeat across sets.

Ex: -random-markers 1000
-random-repeat {number} Choose random number of markers up to the number specified per set. Markers are allowed to repeat across sets.

Ex: -random-repeat 1000
-sets {number} Specify the number of random sets of markers you would like.

Ex: -sets 5
-zero-disabled Zero genotypes of disabled samples in formatting steps (-inc-disabled-samples required)

Ex: -inc-disabled-samples -zero-disabled
-zero-excluded Zero genotypes of excluded samples specified by the command line argument(s) -incsamples or -excsamples (intended to be used with -inc-excluded-samples)

Ex: -inc-excluded-samples -zero-exclude
